ZeroTrust Agents includes native support for the Model Context Protocol, enabling transparent interception of tool calls between applications and MCP servers.

## Supported Transports

The Gateway supports both standard MCP transport options:

1.  **Stdio Transport**: For local development and command-line tool integrations.
2.  **SSE (Server-Sent Events) Transport**: For cloud-native microservices and distributed deployments.

## How MCP Security Works

When an agent calls a tool via the MCP client, the Gateway intercepts the `tools/call` JSON request. It evaluates parameters against policy constraints, then either forwards the call to the actual MCP server using `httpx` or returns a mock/blocked error response to the client.

## Tenant Configuration

Each tenant can configure a dedicated `mcp_upstream_url` via the dashboard. This allows you to host multiple upstream MCP servers mapped to individual tenants.

## Human-in-the-Loop (HITL) for MCP

The Gateway fully supports `require_approval` policies for MCP. If an agent calls a protected tool via MCP, the JSON-RPC request is suspended in memory while a push notification is sent to the administrator. The call will hang until the administrator clicks **Approve** or **Reject**, ensuring total control over the agent's filesystem or database access.
